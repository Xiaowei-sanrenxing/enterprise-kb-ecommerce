# DRI 责任地图（极薄组织上下文）

> 不是部门墙,只回答:每条工作流,谁是那个被命名的负责人(DRI)。
> AI Native 公司不靠"部门",靠"每件事一个 DRI"。

| 工作流 / 域 | DRI | 职责 |
|---|---|---|
| 选品与供应链 | 〔DRI〕 | 选品、货源、备货 |
| 内容与流量 | 〔DRI〕 | 详情页、直播、投放 |
| 销售转化 | 〔DRI〕 | 客服话术、转化、客单价 |
| 履约与售后 | 〔DRI〕 | 发货、退换、评价管理 |
| 复购与私域 | 〔DRI〕 | 会员、复购、社群 |
| 知识库治理 | 〔知识库负责人〕 | 准入/定版/权限/同步 |

> 注:初期可单人多角色,团队扩张后每条工作流的 DRI 拆给具体的人,原则不变。
