# 复购与私域 SOP · 人可读总览

> 给人看的全流程。Agent 执行请走 [`_task-card.md`](_task-card.md)。

## 一句话流程
成交买家沉淀进私域 → 按 RFM 分层 → 各层差异化触达（新客建信任/潜力推二单/沉睡召回/VIP维护）→ 踩消耗周期提醒复购 → 社群价值运营 → 各层复购率回流优化。

## 几条铁律
1. **分层发不群发**：不同层不同内容/优惠，一刀切群发=浪费+骚扰。
2. **先价值后卖货**：首次触达建信任，上来就推销必死。
3. **踩消耗周期**：按品类补货周期提醒复购，转化最高。
4. **社群价值优先**：硬广克制，天天发广告群必死。
5. **隐私+合规**：会员名单/手机/微信/购买记录 🔴 受限，群发经授权不轰炸、不好评返现。

# 会员 RFM 分层运营（★复购精细化的地基）

> 核心：**按 R（最近购买）F（频次）M（金额）分层，不同层不同打法——给所有人发同一套广告 = 浪费 + 骚扰。**
> 拉新成本是复购成本的数倍，把老客分层养好，是利润最稳的来源。

## RFM 三维
- **R 最近一次购买**：越近越活跃。
- **F 购买频次**：越高越忠诚。
- **M 累计金额**：越高越是大客。

## 五层 + 打法
| 层 | 特征 | 核心动作 |
|---|---|---|
| 🌟 高价值（高 R 高 F 高 M）| 常买、买得多、最近还在买 | VIP 专属权益/新品优先/专属客服，重点维护，绝不打扰式群发 |
| 💎 潜力（高 R 低 F）| 最近买过但次数少 | 引导第二单——关联推荐/复购券，养成习惯 |
| ⚠️ 沉睡（低 R 曾活跃）| 以前常买、最近不买了 | 唤醒：召回券/新品提醒/问需求，抢在流失前 |
| 🆕 新客（首单）| 只买过一次 | 首单后跟进体验+引导加私域+复购钩子，决定能否留住 |
| 💤 流失（长期低 R 低 F）| 很久没动静 | 低成本批量唤醒，无效则不再投入 |

## 关键纪律
1. **分层发，不群发**：不同层给不同内容/优惠，避免对高价值客户发垃圾广告。
2. **首单后是关键窗口**：新客首单体验+及时跟进决定留存，别买完就失联。
3. **唤醒要趁早**：客户进入沉睡期就召回，等流失了成本翻倍。

## 隐私红线
- 会员手机号/微信/购买记录 = 🔴 受限，不外发；群发须经授权、不骚扰式轰炸（合规）。

## 用法
分好层 → 用 [[私域复购触达节奏]] 设定每层的触达时机和钩子。

# 私域复购触达节奏（★把一次性买家养成回头客）

> 核心：**把公域买家沉淀到私域，靠'节奏 + 价值'养复购——上来就群发广告 = 自杀式骚扰。**
> 私域是可反复免费触达的自有流量池，复购率的主战场。

## 第一步：沉淀（把人引进私域）
- 包裹卡/客服引导/会员福利钩子，把成交买家加到微信/社群/会员体系。
- 钩子给真实好处（复购券/专属福利/售后保障），不诱导好评返现。

## 第二步：分层触达节奏（按 RFM 层）
配合 [[会员RFM分层运营]] 的层，设触达时机：
| 时机 | 动作 |
|---|---|
| 首单收货后 1-3 天 | 关怀使用+确认满意（先建信任，不推销） |
| 首单后 7-15 天 | 复购钩子：关联品/复购券，引导第二单 |
| 复购周期临近（按品类消耗周期）| 提醒「该补货了」+ 优惠 |
| 沉睡期 | 召回券/新品/问需求唤醒 |
| 大促/新品 | 分层差异化推送，高价值客户优先 |

## 第三步：社群运营（价值优先）
- **内容配比**：价值内容（干货/福利/互动）为主，硬广克制——天天发广告必死。
- 营造场（晒单/答疑/活动），让群「活」着而不是单向广告板。
- 不刷屏轰炸、不诱导好评（合规）。

## 关键纪律
1. **先给价值后卖货**：首次触达建信任，别一上来推销。
2. **按消耗周期提醒复购**：踩准补货时机，转化最高。
3. **节奏感**：触达有间隔有理由，过密=拉黑，过疏=失联。

## 隐私红线
- 私域客户名单/手机/微信/购买记录 = 🔴 受限，不外发、不买卖；群发经授权、不骚扰（合规）。

## 用法
私域是复购引擎。配合 RFM 分层精准触达，把售后工作流回头的客户接住养成长期复购。

# 第三方代运营 / SCRM 个人信息委托边界（★守住"不裸导出"红线后，给合规共享的边界）

> 补盲区：[[会员RFM分层运营]] 已明确「会员手机号/购买记录=🔴受限，不外发、不买卖」，但只能一刀切说"不能发"。
> 现实里**代运营、SCRM、外包群发**极普遍。本流程给出"怎样才算合规委托第三方处理个人信息"的边界——agent 不裸导出，但能指出合规通路。

## 三句话原则（《个人信息保护法》口径）
1. **裸导出明文手机号丢给第三方群发 = 红线**，无论是否"自家代运营"，agent 一律拒绝直接外发。
2. 合规共享走的是**"委托处理"**，不是"把数据给出去随便用"——受托方只能在我方约定的目的/方式内处理，不得转用、转卖、超范围。
3. 三个硬门槛缺一不可：**单独同意 + 数据处理协议(DPA) + 最小必要**。

## 合规委托第三方处理的硬门槛
| 门槛 | 该怎么做 |
|---|---|
| **单独同意** | 会员须就"向第三方共享个人信息用于营销触达"获得**单独、明示同意**（不是混在总隐私条款里），可撤回；无同意的会员不得纳入共享范围 |
| **数据处理协议(DPA)** | 与代运营/SCRM 签订书面委托处理协议：约定处理目的、方式、期限、保密义务、安全措施、禁止转委托/转售、到期删除/返还、违约与审计权 |
| **最小必要** | 只提供完成触达**必需的最少字段**，能脱敏/打标签触达就不给明文；优先走**不外传明文**的方式（见下）|
| **受托方资质与安全** | 评估第三方数据安全能力，约定加密传输/存储、访问留痕；我方对受托方处理行为承担监督责任 |

## 优先级：能不外传明文就不外传
1. **首选**：在我方〔会员/CRM 系统〕或 SCRM 内**由系统侧调用群发**，第三方只操作界面、拿不到明文导出表（数据不离我方/平台环境）。
2. 次选：脱敏/Hash 后做人群包匹配触达（如平台 ID 加密匹配），不传明文手机号。
3. 仅当确有必要传明文，且**单独同意 + DPA + 最小必要 + 安全措施全部满足**，由〔合规 DRI〕+ 法务审批后，按最小范围、加密方式提供，留痕可审计。

## 🔴 agent 绝不做
- 不导出明文会员手机号/微信/购买记录表格外发给任何第三方
- 不在缺单独同意或无 DPA 的情况下共享会员个人信息
- 不把"自家长期合作代运营"当作可跳过合规门槛的理由
- 不买卖、不超目的使用、不默许第三方转委托/转售

## agent 标准回应（老板说"把会员手机号导表发给代运营群发"时）
"直接导明文手机号发给第三方是隐私红线，我不能这么做。但代运营群发是可以合规做的：① 优先让代运营在我们的〔CRM/SCRM 系统〕里操作，由系统发，数据不导出；② 必须先拿到会员对'向第三方共享做营销'的单独同意，没同意的不能纳入；③ 和代运营签数据处理协议(DPA)，约定只为这次触达用、不得转售/转委托、到期删除；④ 字段最小必要，能脱敏匹配就不给明文。这几条齐了再由〔合规 DRI〕+ 法务过一遍。要不要我先帮你列同意话术和 DPA 要点清单？"

## 需人工确认 / 上报
- 任何向第三方共享会员个人信息的方案 → 〔合规 DRI〕+ 法务审批，不自动执行
- 明文外传 → 零自动执行，必须人工签批留痕

# 权限分级 · 四级

> 企业库和个人库最硬的区别。没有权限治理 = AI 资料垃圾场 + 隐私事故。

| 级别 | 含义 | 典型资产 | agent 行为 |
|---|---|---|---|
| 🟢 公开 | 可对外发布 | 对外物料、方法论、介绍 | 可写入对外产物 |
| 🟡 内部 | 团队内用 | 话术、SOP、运营数据 | 内部可用,不对外 |
| 🔴 受限 | 隐私/敏感 | 客户对话、通话记录、手机号、名单 | **绝不写入对外产物,只内部分析** |
| ⛔ 禁止入库 | 不该进库 | 证件、银行信息、未授权第三方资料 | 拒绝入库 |

## agent 红线
1. 🔴 受限资产 → 输出时必须脱敏/打码,绝不原文外发
2. 超标准的报价/优惠/对外承诺 → 必须人工确认
3. 不确定权限 → 默认按更高一级处理,问用户

# 电商行业 · 合规与权限默认值

> 合成进成品库 `90-governance/行业合规默认.md`。客户可按自身情况收紧。

## 默认权限分级（通用四级 + 行业补充）
| 资产类型 | 默认级别 | 说明 |
|---|---|---|
| 客户聊天原文 / 订单信息 / 收货地址 / 手机号 / 微信号 | 🔴 受限 | 绝不写入任何对外产物，绝不进飞书，引用必脱敏 |
| 供应商报价 / 进货成本 / 利润结构 | 🔴 受限 | 核心机密，仅授权人可读 |
| 内部话术 / SOP / 运营数据 / 投放后台数据 | 🟡 内部 | 团队内用，不对外 |
| 详情页文案 / 商品卖点 / 品牌介绍 | 🟢 公开 | 对外物料 |
| 未脱敏的真实客户案例/评价截图 | 🔴 受限 | 脱敏后方可升公开 |

## 行业红线（agent 触碰即停，转人工确认）

### 1. 虚假宣传 / 极限词
- 禁用「最」「第一」「国家级」「顶级」「绝无仅有」「100%」等绝对化、极限词（《广告法》明令禁止）。
- 不得编造功效、夸大效果、虚构成分/产地/资质；保健/医疗类不得宣称疗效。
- 不得伪造「销量/好评/获奖/权威认证」等背书。
- agent 写文案/详情页一律走「事实可验证」原则，拿不准的形容词不写。

### 2. 价格欺诈
- 禁止虚构原价/划线价（划线价必须是近期真实成交价）。
- 禁止「先涨后降」假打折、虚假满减、误导性赠品。
- 标价与实付一致，不得有隐藏费用。

### 3. 七天无理由退货
- 法定支持七天无理由（生鲜/定制/拆封贴身用品等法定除外类目除外，且须事先明示）。
- 不得擅自拒绝合理无理由退货、不得设置不合理退货门槛。
- 售后 agent 默认按「支持七天无理由」处理，例外类目须有明示依据。

### 4. 刷单 / 虚假交易（绝对红线）
- 严禁刷单、刷好评、删差评、虚构物流、组织炒信。
- 严禁诱导/胁迫好评（如「好评返现」属违规）、严禁恶意操纵评价。
- agent 不得生成任何刷单/控评相关内容或话术，遇到此类需求直接拒绝并提示风险。

### 5. 消费者权益
- 不得泄露、买卖客户个人信息（订单/地址/手机号）。
- 不得在客户未同意下骚扰式营销（短信/电话轰炸）。
- 退换货、发票、质量问题须依法履约，不得推诿。

### 6. 平台规则
- 遵守所在平台（淘系/抖音/拼多多/京东等）的类目资质、发布、营销、物流时效规则。
- 不得跨平台引流违规、不得绕过平台交易（私下转账规避保障）。
- 类目准入资质（如食品经营许可/品牌授权）须齐全方可经营。

## 权限分级处置原则
- agent 默认按「最严」处理：拿不准级别 → 当受限处理 → 问负责人。
- 涉及客户隐私、超授权报价/优惠、价格/宣传合规判断 → 须人工确认，不自动执行。
- 任何对外产物（详情页/朋友圈/案例/物料）发布前过一遍本红线清单。
