# 任务卡 · 选品与供应链（product-supply）

> Agent 操作总台。接到"评估新品/定货源/算备货/巡库存"类任务，从这里开始。

## 🎯 目标(Goal)
把一个候选品，从评估推进到「上/不上」的决策，定好货源与首批备货量，并维持库存健康——全程不踩三大坑（拍脑袋选品/负毛利硬上/断货掉权重）。

## 📥 上下文(Context) —— 动手前必读全
1. [`context/选品三维评估模型.md`](context/选品三维评估模型.md) — 需求×竞争×利润打分定生死
2. [`context/备货与库存健康线.md`](context/备货与库存健康线.md) — 备多少、什么时候补、怎么清滞销
3. 〔货源/供应商库〕 — 报价、起订量、工期（真实库含，🔴受限）
4. 〔类目数据工具〕 — 搜索量/竞争/趋势（真实库接生意参谋等）

## 🪜 指令(Instructions)
1. 拉候选品的需求/竞争/利润数据 → 过三维模型打分
2. 任一轴 <3 或毛利 <20% → 毙掉或提改造方案（换货源/找差异化）
3. 过关品 → 比价定货源（核对资质/起订量/工期）
4. 按备货公式算首批量（新品小批试，别一把梭）
5. 上架后每日巡库存健康线 → 预警即补、断货风险降速、滞销果断清
6. 每个决策记录依据与结果 → 回流复盘

## 🛠 工具(Tools)
〔类目数据工具〕 · 〔货源/供应商库（受限）〕 · 〔库存/ERP 读写〕 · 三维打分模板

## 🔒 权限(Permission)
- 可读：选品模型、备货规则、公开类目数据
- 🔴 受限：供应商报价/进货成本/利润结构 → 仅授权人，绝不外发
- 需人工确认：大额首单备货、新供应商合作、清仓甩货定价

## ✅ 验收(Acceptance)
- 每个上架决策有三维打分依据 ｜ 毛利<20% 未硬上 ｜ 库存无意外断货/超期滞销 ｜ 成本数据未外泄
- 详见 evals.md

## 🔄 自我进化接口
每个品记录"打分 → 实际表现（爆/平/滞销/退货率）" → 回流校准三维模型阈值 → 下次选品更准。

# 选品与供应链 SOP · 人可读总览

> 给人看的全流程。Agent 执行请走 [`_task-card.md`](_task-card.md)。

## 一句话流程
拉数据 → 三维打分定生死 → 过关品比价定货源 → 小批试备货 → 每日巡库存健康线 → 记录结果回流。

## 几条铁律
1. **不拍脑袋选品**：需求/竞争/利润三轴同时过关才上，拍脑袋是积压头号原因。
2. **负毛利不碰**：毛利<20% 再火也不上，投流+退货+履约一摊就亏。
3. **新品小批试**：没历史数据先备一个补货周期的量，别一把梭。
4. **断货比压货更伤**：售罄掉排名，宁可小步勤补；滞销则果断止损清仓。
5. **成本红线**：供应商报价/进货成本 🔴 受限，绝不外发。

## 详细资产
- [`context/选品三维评估模型.md`](context/选品三维评估模型.md) — 三轴打分定生死
- [`context/备货与库存健康线.md`](context/备货与库存健康线.md) — 备货公式 + 库存红绿灯
- 〔货源/供应商库〕 — 客户补：报价/起订量/工期（受限）

# 备货与库存健康线（★压货 vs 断货的平衡）

> 核心：**备货 = 日均销量 × (补货周期 + 安全天数)；断货丢排名，压货占现金，两头都是坑。**

## 备货公式
```
建议备货量 = 预估日均销量 × (补货提前期 + 安全库存天数)
安全库存天数 = 补货提前期 × 0.3~0.5（越不稳越高）
```
- 新品无历史 → 用同类品/测款期数据估日均，首单**小批量试**（够卖一个补货周期即可），别一把梭。
- 爆款/大促前 → 按预估峰值上调，预留缓冲。

## 库存健康线（红绿灯，agent 每日巡检）
| 状态 | 可售天数（现货 ÷ 日均销量） | 动作 |
|---|---|---|
| 🟢 健康 | 补货周期 + 安全天数 以上 | 维持 |
| 🟡 预警 | 接近补货周期 | 立刻下补货单 |
| 🔴 断货风险 | < 补货周期 | 紧急补货 + 必要时降速投流/限流，避免售罄掉权重 |
| ⚫ 滞销 | 周转天数远超均值（如 >90 天） | 清仓：捆绑/降价/达人甩货，回笼现金 |

## 关键纪律
1. **断货比压货更伤**：售罄掉自然排名，恢复要重新养，宁可小步勤补。
2. **滞销要果断止损**：占现金占仓储，越拖越亏，按周扫滞销清单。
3. **补货周期要算到真实到货**（含生产+物流+质检），别只算供应商口头工期。

## 数据
日均销量、周转天数、货源报价均属 🔴 受限运营/成本数据，仅内部用。

# 选品三维评估模型（★选品第一道闸）

> 核心：**需求 × 竞争 × 利润，三轴同时过关才上；任一轴红线直接毙。**
> 拍脑袋选品是亏货积压的头号原因，所有候选品先过这张表。

## 三维打分（每轴 1-5 分，<3 即红线）

| 维度 | 看什么 | 5 分（强上） | 1 分（毙掉） |
|---|---|---|---|
| **需求** | 搜索量/增长趋势/季节性/受众规模 | 需求大且上升、非强季节 | 需求小或断崖下滑、过季 |
| **竞争** | 在售店铺数/头部垄断度/同质化 | 蓝海、头部未垄断、有差异化空间 | 红海、被大店锁死、纯比价 |
| **利润** | 毛利率/退货率/履约成本 | 毛利≥40%、退货率低 | 毛利<20% 或退货率高 |

> 利润轴红线：**毛利 < 20% 基本不碰**（投流+退货+履约一摊就亏）；需求或竞争任一 <2 也毙。

## 判定规则
- 三轴均 ≥3 且总分 ≥11 → 进测款池
- 任一轴 <3 → 毙掉或改造（换货源降本/找差异化避竞争）
- 利润轴 <2 → 无条件毙（再火的需求扛不住负毛利）

## 数据来源（拿不准的填占位，客户接自己的工具）
- 需求：〔平台生意参谋/蝉妈妈/搜索指数〕
- 竞争：〔类目在售数/竞品销量分布〕
- 利润：〔货源报价（受限）+ 退货率历史 + 履约成本〕

## 用法
选品第一步先打分定生死，过关的品才进 [[备货与库存健康线]] 算备多少。
利润轴里的货源成本属 🔴 受限数据，不得外发。

# 权限分级 · 四级

> 企业库和个人库最硬的区别。没有权限治理 = AI 资料垃圾场 + 隐私事故。

| 级别 | 含义 | 典型资产 | agent 行为 |
|---|---|---|---|
| 🟢 公开 | 可对外发布 | 对外物料、方法论、介绍 | 可写入对外产物 |
| 🟡 内部 | 团队内用 | 话术、SOP、运营数据 | 内部可用,不对外 |
| 🔴 受限 | 隐私/敏感 | 客户对话、通话记录、手机号、名单 | **绝不写入对外产物,只内部分析** |
| ⛔ 禁止入库 | 不该进库 | 证件、银行信息、未授权第三方资料 | 拒绝入库 |

## agent 红线
1. 🔴 受限资产 → 输出时必须脱敏/打码,绝不原文外发
2. 超标准的报价/优惠/对外承诺 → 必须人工确认
3. 不确定权限 → 默认按更高一级处理,问用户

## 默认权限分级（通用四级 + 行业补充）
| 资产类型 | 默认级别 | 说明 |
|---|---|---|
| 客户聊天原文 / 订单信息 / 收货地址 / 手机号 / 微信号 | 🔴 受限 | 绝不写入任何对外产物，绝不进飞书，引用必脱敏 |
| 供应商报价 / 进货成本 / 利润结构 | 🔴 受限 | 核心机密，仅授权人可读 |
| 内部话术 / SOP / 运营数据 / 投放后台数据 | 🟡 内部 | 团队内用，不对外 |
| 详情页文案 / 商品卖点 / 品牌介绍 | 🟢 公开 | 对外物料 |
| 未脱敏的真实客户案例/评价截图 | 🔴 受限 | 脱敏后方可升公开 |

## 行业红线（agent 触碰即停，转人工确认）

### 1. 虚假宣传 / 极限词
- 禁用「最」「第一」「国家级」「顶级」「绝无仅有」「100%」等绝对化、极限词（《广告法》明令禁止）。
- 不得编造功效、夸大效果、虚构成分/产地/资质；保健/医疗类不得宣称疗效。
- 不得伪造「销量/好评/获奖/权威认证」等背书。
- agent 写文案/详情页一律走「事实可验证」原则，拿不准的形容词不写。

### 2. 价格欺诈
- 禁止虚构原价/划线价（划线价必须是近期真实成交价）。
- 禁止「先涨后降」假打折、虚假满减、误导性赠品。
- 标价与实付一致，不得有隐藏费用。

### 3. 七天无理由退货
- 法定支持七天无理由（生鲜/定制/拆封贴身用品等法定除外类目除外，且须事先明示）。
- 不得擅自拒绝合理无理由退货、不得设置不合理退货门槛。
- 售后 agent 默认按「支持七天无理由」处理，例外类目须有明示依据。

### 4. 刷单 / 虚假交易（绝对红线）
- 严禁刷单、刷好评、删差评、虚构物流、组织炒信。
- 严禁诱导/胁迫好评（如「好评返现」属违规）、严禁恶意操纵评价。
- agent 不得生成任何刷单/控评相关内容或话术，遇到此类需求直接拒绝并提示风险。

### 5. 消费者权益
- 不得泄露、买卖客户个人信息（订单/地址/手机号）。
- 不得在客户未同意下骚扰式营销（短信/电话轰炸）。
- 退换货、发票、质量问题须依法履约，不得推诿。

### 6. 平台规则
- 遵守所在平台（淘系/抖音/拼多多/京东等）的类目资质、发布、营销、物流时效规则。
- 不得跨平台引流违规、不得绕过平台交易（私下转账规避保障）。
- 类目准入资质（如食品经营许可/品牌授权）须齐全方可经营。

## 权限分级处置原则
- agent 默认按「最严」处理：拿不准级别 → 当受限处理 → 问负责人。
- 涉及客户隐私、超授权报价/优惠、价格/宣传合规判断 → 须人工确认，不自动执行。
- 任何对外产物（详情页/朋友圈/案例/物料）发布前过一遍本红线清单。
