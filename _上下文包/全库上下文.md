# 内容与流量 SOP · 人可读总览

> 给人看的全流程。Agent 执行请走 [`_task-card.md`](_task-card.md)。

## 一句话流程
搭承接（详情页/直播按三屏三段）→ 过合规自检 → 查承接力达标 → 小预算测款 → 三关过再放量 → 守 ROI/库存 → 沉淀爆款结构。

## 几条铁律
1. **先承接后放量**：转化承接力不达标就加投流 = 放大亏损。
2. **卖点说好处不说参数**：「出门一天不断电」胜过「2000mAh」。
3. **测款再放量**：新品小预算测点击/转化/ROI，三关过才加钱。
4. **ROI 守线**：持续低于盈亏平衡点就止损，不为冲量硬亏。
5. **合规红线**：禁极限词/虚假功效/虚构原价，评价脱敏、禁刷单控评。

# 流量结构与投放红线（★花钱买量前必读）

> 核心：**先有转化承接力，再放量；ROI<1 持续投 = 烧钱给平台。**
> 流量分自然+付费两条腿，付费是放大器——放大的是转化效率，承接力差时放大的是亏损。

## 流量结构（三条腿）
| 来源 | 特点 | 打法 |
|---|---|---|
| 自然流量 | 免费、靠权重/排名 | 养：销量+好评+DSR 推坑产，长期地基 |
| 付费投流 | 花钱、可控、即时 | 直通车/引力魔方/千川等，按 ROI 调 |
| 内容/直播 | 短视频/直播间种草 | 内容种草 + 直播逼单，导流回详情页/链接 |

## 投放红线（agent 投放前自检）
1. **承接力优先**：详情页转化率/直播间转化没达健康线（如 <行业基准）→ 先优化承接，别加投流。负毛利或转化太差时放量=放大亏损。
2. **小预算测款再放量**：新品先小预算测点击率/转化率/ROI，跑出正反馈再加预算，不一上来梭哈。
3. **ROI 守线**：算清盈亏平衡 ROI（含毛利/退货），持续 <平衡点要止损或优化，不为冲量硬亏。
4. **断货不投**：库存不足时降速/暂停投放（见选品供应链），避免售罄掉权重。

## 测款节奏
```
小预算测 → 看点击率(够不够吸引) → 看转化率(承接行不行) → 看 ROI(赚不赚)
三关都过 → 逐步加预算放量；任一关挂 → 优化对应环节，不盲目加钱
```

## 直播/内容要点
- 直播间也是「三屏逻辑」：开场抓人、中段说服、逼单促成。
- 投流千川等同理：先测内容素材，跑出爆款素材再放量。
- 一切宣传遵守合规红线（禁极限词/虚假功效/虚构价格）。

## 数据
投放后台数据、ROI、成本属 🟡 内部，不对外。

# 详情页黄金三屏（★转化主战场）

> 核心：**首屏抓人（3 秒留住）→ 中屏说服（解决疑虑）→ 尾屏逼单（临门一脚）。**
> 转化率低，80% 是详情页结构没搭对，不是流量不够。

## 三屏结构

### 首屏 · 抓人（决定跳出率）
- 一句话讲清「卖给谁 + 解决什么问题 + 凭什么是你」。
- 主图突出核心卖点/使用场景，不堆字、不放无关美图。
- 3 秒内让访客判断「跟我有没有关」，没说清就跳走。

### 中屏 · 说服（决定加购）
- 卖点场景化：把参数翻译成「对你的好处」（不是「容量2000mAh」而是「出门一天不断电」）。
- 打消疑虑：材质/工艺/实测/对比/资质（事实可验证，禁极限词，见合规默认）。
- 信任背书：真实评价/销量/使用前后对比（脱敏，禁刷单控评）。

### 尾屏 · 逼单（决定下单）
- 临门一脚：限时优惠/赠品/无理由退货承诺/物流时效。
- 降低决策成本：七天无理由、运费险、售后保障说清楚。
- 不虚构原价/不先涨后降（价格合规红线）。

## 自检清单（发布前过）
- [ ] 首屏 3 秒能看懂卖给谁、解决什么
- [ ] 卖点是"好处"不是"参数"
- [ ] 无极限词/无虚假功效/无虚构原价
- [ ] 评价/案例已脱敏，无刷单痕迹
- [ ] 尾屏有明确下单理由 + 售后保障

## 用法
详情页是承接所有流量的落点。投放/直播引来的流量，最终都靠它转化，先把它搭对再加 [[流量结构与投放红线]] 的量。

# 履约与售后 SOP · 人可读总览

## 一句话流程
按时效发货+异常预警 → 售后先共情判类型 → 依法快给方案 → 每笔退货归因回流 → 中差评真诚处理争取修改 → 异常拉根因。

## 几条铁律
1. **依法快处不推诿**：七天无理由默认通过，质量问题优先换退担运费。
2. **退货必归因**：白退一单不可惜，可惜的是不知道为什么退、问题继续发生。
3. **绝不刷单删评**：差评靠真诚解决和优化治本，刷单/删评/胁迫改评是平台高压红线。
4. **绝不好评返现**：属违规营销。
5. **隐私红线**：买家订单/地址/手机/聊天 🔴 受限，绝不外泄，归因记录脱敏。

# 恶意 / 同行差评 合规申诉举报路径（★守住"不删评"红线后，给正向出路）

> 补盲区：[[退货归因与评价管理]] 已明确「不得删差评/不得利诱改评」红线，但只能答"按平台规则处理"。
> 本流程专管**疑似恶意、同行攻击、敲诈勒索类差评**——agent 不删评、不对线，但要给出**合规的申诉/举报正向通道**。

## 第一判断：是哪一类差评（决定走哪条路）
| 类型 | 信号 | 走向 |
|---|---|---|
| 真实负评 | 买家真买了、真有体验问题 | 走 [[退货归因与评价管理]]：真诚联系解决 + 回流改进，**不申诉** |
| **疑似恶意/同行** | 未购买却评、批量小号、内容与商品无关、纯人身攻击、明显抹黑 | 走本流程，**走平台申诉** |
| **敲诈勒索** | "不退款/不赔钱就继续差评""给我返现就删评" | 走本流程，**截图固证 + 平台举报 + 必要时报警/法务** |

## 处置四步（agent 协助整理材料，不自行删评/对线）
1. **取证固定**：第一时间截图/录屏差评内容（评价时间、买家账号、订单号或"无订单"证据、聊天威胁记录），不删不公开回怼。
2. **判定理由对号入座**：按平台恶意评价申诉常见受理理由归类——
   - 未发生真实交易（无对应订单）
   - 内容与商品/服务无关、纯广告或人身攻击
   - 同一买家/小号批量恶意评价
   - 含敲诈勒索、违法违规信息
   - 泄露他人隐私或含不实诋毁
3. **走平台官方申诉/举报通道**（由运营/售后主管提交，agent 整理材料）：
   - 商家后台「评价管理 / 评价申诉」入口提交申诉，附第 1 步证据
   - 平台「不正当竞争 / 恶意评价举报」通道（针对同行水军）
   - 敲诈勒索类：保留证据 → 平台举报 + 必要时报警，转法务评估
4. **正向回应**：申诉期间可在差评下做**克制、就事论事的公开回复**（说明已联系/已申诉、不攻击买家），避免二次发酵。

## 🔴 agent 绝不做
- 不自行删差评、不找平台/第三方"刷掉"差评（属 [[退货归因与评价管理]] 绝对红线）
- 不利诱/胁迫买家改评或撤评（"返现删评"既违规又可能正中敲诈圈套）
- 不在评论区和买家/同行对线互撕、不捏造反击内容
- 不擅自把差评定性为"恶意"对外宣称——定性由平台申诉裁定，对外措辞从严

## agent 标准回应（老板问"这条恶意差评能不能删"时）
"删差评是违规红线，我不能删，也不能找人刷掉。但这条像同行恶意/敲诈，可以走正向通道：① 我先把证据截图固证（时间、账号、有无真实订单、威胁记录）；② 按'未真实交易/与商品无关/敲诈勒索'对号申诉，由〔售后主管〕在评价管理后台提交平台申诉；③ 如属同行水军走不正当竞争举报，敲诈勒索还要留证报警转法务。材料我已帮你整理好，等你确认提交。"

## 需人工确认 / 上报
- 申诉提交、对外公开回复定稿 → 〔售后主管〕确认
- 敲诈勒索、批量水军、涉法律 → 上报〔合规 DRI〕+ 法务

# 退换货处理标准流程（★售后核心 + 合规高压区）

> 核心：**依法快处、息事宁人、把退货变复购机会——不推诿、不违规拒退。**
> 售后处理得好，差评变好评、退货客变回头客；处理砸了，差评+投诉+平台扣分三连。

## 退换货处理四步

### 1. 接诉 + 判类型
| 类型 | 处理原则 |
|---|---|
| 七天无理由 | 法定支持，默认通过（法定除外类目须事先明示才可拒，见合规） |
| 质量问题 | 优先换/退 + 必要补偿，承担来回运费 |
| 物流问题（破损/丢件）| 查物流凭证，补发/退款，向承运方追责 |
| 错发/漏发 | 立即补发/退差，态度先认错 |
| 恶意/异常退货 | 留证据，按平台规则处理，不擅自定性 |

### 2. 快速响应 + 安抚
- 先共情认问题，再给方案，别先甩责任。
- 不推诿、不设不合理门槛（合规红线）。

### 3. 给方案 + 闭环
- 能退/换/补就利落给，必要时小额补偿挽留。
- 超授权补偿/特殊处理 → 人工确认。

### 4. 归因记录（接售后归因闭环）
- 每笔退货记真实原因 → 回流（见 [[退货归因与评价管理]]）。

## 合规红线（绝对不可碰）
- **不得违规拒绝七天无理由**（法定除外类目须事先明示）。
- **不得拿处理售后胁迫/交换删差评**（如「退款可以，但你得删差评」属违规）。
- **不得诱导好评返现**。
- **不得拖延/推诿法定售后义务。**

## 隐私红线
- 买家订单信息/收货地址/手机号 = 🔴 受限，处理过程不外泄，记录归因时脱敏。

## 用法
售后是履约的兜底。处理完务必把退货原因喂给 [[退货归因与评价管理]] 形成改进闭环。

# 退货归因与评价管理（★把售后数据变成改进燃料）

> 核心：**每笔退货都是一条改进线索，每条差评都是一次挽回+优化机会。绝不靠刷单控评掩盖。**

## 退货归因闭环
每笔退货记真实原因，按类回流到对应工作流：
| 退货原因 | 回流到哪 |
|---|---|
| 质量/做工差 | → 选品供应链（换货源/加质检） |
| 与描述不符 | → 内容流量（修详情页，停止夸大） |
| 尺码/颜色不对 | → 详情页（补尺码表/实拍色） |
| 物流慢/破损 | → 履约（换承运/加固包装） |
| 不喜欢/拍错 | → 正常无理由，关注是否冲动消费引导过度 |

**用法**：退货率升高 → 拉归因分布 → 找占比最高的原因 → 回流对应工作流修，而不是只盯客服。

## 评价管理（合规高压区）
### 标准动作
1. **好评**：感谢，可引导追评/晒图（不返现、不诱导，合规）。
2. **中差评**：第一时间真诚联系买家了解问题 → 解决 → 自然争取修改（不胁迫）。
3. **沉淀**：差评反映的真问题回流改进，比删评更治本。

### 绝对红线（碰即违规）
- **不得刷好评、刷销量、虚假交易炒信。**
- **不得删差评、不得胁迫/利诱买家改差评。**
- **不得好评返现。**
- agent 遇到「帮忙刷单/控评/删差评」类需求 → 直接拒绝并提示风险。

## 数据
退货归因数据、评价数据属 🟡 内部；买家身份信息 🔴 受限脱敏。

# 备货与库存健康线（★压货 vs 断货的平衡）

> 核心：**备货 = 日均销量 × (补货周期 + 安全天数)；断货丢排名，压货占现金，两头都是坑。**

## 备货公式
```
建议备货量 = 预估日均销量 × (补货提前期 + 安全库存天数)
安全库存天数 = 补货提前期 × 0.3~0.5（越不稳越高）
```
- 新品无历史 → 用同类品/测款期数据估日均，首单**小批量试**（够卖一个补货周期即可），别一把梭。
- 爆款/大促前 → 按预估峰值上调，预留缓冲。

## 库存健康线（红绿灯，agent 每日巡检）
| 状态 | 可售天数（现货 ÷ 日均销量） | 动作 |
|---|---|---|
| 🟢 健康 | 补货周期 + 安全天数 以上 | 维持 |
| 🟡 预警 | 接近补货周期 | 立刻下补货单 |
| 🔴 断货风险 | < 补货周期 | 紧急补货 + 必要时降速投流/限流，避免售罄掉权重 |
| ⚫ 滞销 | 周转天数远超均值（如 >90 天） | 清仓：捆绑/降价/达人甩货，回笼现金 |

## 关键纪律
1. **断货比压货更伤**：售罄掉自然排名，恢复要重新养，宁可小步勤补。
2. **滞销要果断止损**：占现金占仓储，越拖越亏，按周扫滞销清单。
3. **补货周期要算到真实到货**（含生产+物流+质检），别只算供应商口头工期。

## 数据
日均销量、周转天数、货源报价均属 🔴 受限运营/成本数据，仅内部用。

# 选品三维评估模型（★选品第一道闸）

> 核心：**需求 × 竞争 × 利润，三轴同时过关才上；任一轴红线直接毙。**
> 拍脑袋选品是亏货积压的头号原因，所有候选品先过这张表。

## 三维打分（每轴 1-5 分，<3 即红线）

| 维度 | 看什么 | 5 分（强上） | 1 分（毙掉） |
|---|---|---|---|
| **需求** | 搜索量/增长趋势/季节性/受众规模 | 需求大且上升、非强季节 | 需求小或断崖下滑、过季 |
| **竞争** | 在售店铺数/头部垄断度/同质化 | 蓝海、头部未垄断、有差异化空间 | 红海、被大店锁死、纯比价 |
| **利润** | 毛利率/退货率/履约成本 | 毛利≥40%、退货率低 | 毛利<20% 或退货率高 |

> 利润轴红线：**毛利 < 20% 基本不碰**（投流+退货+履约一摊就亏）；需求或竞争任一 <2 也毙。

## 判定规则
- 三轴均 ≥3 且总分 ≥11 → 进测款池
- 任一轴 <3 → 毙掉或改造（换货源降本/找差异化避竞争）
- 利润轴 <2 → 无条件毙（再火的需求扛不住负毛利）

## 数据来源（拿不准的填占位，客户接自己的工具）
- 需求：〔平台生意参谋/蝉妈妈/搜索指数〕
- 竞争：〔类目在售数/竞品销量分布〕
- 利润：〔货源报价（受限）+ 退货率历史 + 履约成本〕

## 用法
选品第一步先打分定生死，过关的品才进 [[备货与库存健康线]] 算备多少。
利润轴里的货源成本属 🔴 受限数据，不得外发。

# 会员 RFM 分层运营（★复购精细化的地基）

> 核心：**按 R（最近购买）F（频次）M（金额）分层，不同层不同打法——给所有人发同一套广告 = 浪费 + 骚扰。**
> 拉新成本是复购成本的数倍，把老客分层养好，是利润最稳的来源。

## RFM 三维
- **R 最近一次购买**：越近越活跃。
- **F 购买频次**：越高越忠诚。
- **M 累计金额**：越高越是大客。

## 五层 + 打法
| 层 | 特征 | 核心动作 |
|---|---|---|
| 🌟 高价值（高 R 高 F 高 M）| 常买、买得多、最近还在买 | VIP 专属权益/新品优先/专属客服，重点维护，绝不打扰式群发 |
| 💎 潜力（高 R 低 F）| 最近买过但次数少 | 引导第二单——关联推荐/复购券，养成习惯 |
| ⚠️ 沉睡（低 R 曾活跃）| 以前常买、最近不买了 | 唤醒：召回券/新品提醒/问需求，抢在流失前 |
| 🆕 新客（首单）| 只买过一次 | 首单后跟进体验+引导加私域+复购钩子，决定能否留住 |
| 💤 流失（长期低 R 低 F）| 很久没动静 | 低成本批量唤醒，无效则不再投入 |

## 关键纪律
1. **分层发，不群发**：不同层给不同内容/优惠，避免对高价值客户发垃圾广告。
2. **首单后是关键窗口**：新客首单体验+及时跟进决定留存，别买完就失联。
3. **唤醒要趁早**：客户进入沉睡期就召回，等流失了成本翻倍。

## 隐私红线
- 会员手机号/微信/购买记录 = 🔴 受限，不外发；群发须经授权、不骚扰式轰炸（合规）。

# 私域复购触达节奏（★把一次性买家养成回头客）

> 核心：**把公域买家沉淀到私域，靠'节奏 + 价值'养复购——上来就群发广告 = 自杀式骚扰。**
> 私域是可反复免费触达的自有流量池，复购率的主战场。

## 第一步：沉淀（把人引进私域）
- 包裹卡/客服引导/会员福利钩子，把成交买家加到微信/社群/会员体系。
- 钩子给真实好处（复购券/专属福利/售后保障），不诱导好评返现。

## 第二步：分层触达节奏（按 RFM 层）
配合 [[会员RFM分层运营]] 的层，设触达时机：
| 时机 | 动作 |
|---|---|
| 首单收货后 1-3 天 | 关怀使用+确认满意（先建信任，不推销） |
| 首单后 7-15 天 | 复购钩子：关联品/复购券，引导第二单 |
| 复购周期临近（按品类消耗周期）| 提醒「该补货了」+ 优惠 |
| 沉睡期 | 召回券/新品/问需求唤醒 |
| 大促/新品 | 分层差异化推送，高价值客户优先 |

## 第三步：社群运营（价值优先）
- **内容配比**：价值内容（干货/福利/互动）为主，硬广克制——天天发广告必死。
- 营造场（晒单/答疑/活动），让群「活」着而不是单向广告板。
- 不刷屏轰炸、不诱导好评（合规）。

## 关键纪律
1. **先给价值后卖货**：首次触达建信任，别一上来推销。
2. **按消耗周期提醒复购**：踩准补货时机，转化最高。
3. **节奏感**：触达有间隔有理由，过密=拉黑，过疏=失联。

## 隐私红线

# 第三方代运营 / SCRM 个人信息委托边界（★守住"不裸导出"红线后，给合规共享的边界）

> 补盲区：[[会员RFM分层运营]] 已明确「会员手机号/购买记录=🔴受限，不外发、不买卖」，但只能一刀切说"不能发"。
> 现实里**代运营、SCRM、外包群发**极普遍。本流程给出"怎样才算合规委托第三方处理个人信息"的边界——agent 不裸导出，但能指出合规通路。

## 三句话原则（《个人信息保护法》口径）
1. **裸导出明文手机号丢给第三方群发 = 红线**，无论是否"自家代运营"，agent 一律拒绝直接外发。
2. 合规共享走的是**"委托处理"**，不是"把数据给出去随便用"——受托方只能在我方约定的目的/方式内处理，不得转用、转卖、超范围。
3. 三个硬门槛缺一不可：**单独同意 + 数据处理协议(DPA) + 最小必要**。

## 合规委托第三方处理的硬门槛
| 门槛 | 该怎么做 |
|---|---|
| **单独同意** | 会员须就"向第三方共享个人信息用于营销触达"获得**单独、明示同意**（不是混在总隐私条款里），可撤回；无同意的会员不得纳入共享范围 |
| **数据处理协议(DPA)** | 与代运营/SCRM 签订书面委托处理协议：约定处理目的、方式、期限、保密义务、安全措施、禁止转委托/转售、到期删除/返还、违约与审计权 |
| **最小必要** | 只提供完成触达**必需的最少字段**，能脱敏/打标签触达就不给明文；优先走**不外传明文**的方式（见下）|
| **受托方资质与安全** | 评估第三方数据安全能力，约定加密传输/存储、访问留痕；我方对受托方处理行为承担监督责任 |

## 优先级：能不外传明文就不外传
1. **首选**：在我方〔会员/CRM 系统〕或 SCRM 内**由系统侧调用群发**，第三方只操作界面、拿不到明文导出表（数据不离我方/平台环境）。
2. 次选：脱敏/Hash 后做人群包匹配触达（如平台 ID 加密匹配），不传明文手机号。
3. 仅当确有必要传明文，且**单独同意 + DPA + 最小必要 + 安全措施全部满足**，由〔合规 DRI〕+ 法务审批后，按最小范围、加密方式提供，留痕可审计。

## 🔴 agent 绝不做
- 不导出明文会员手机号/微信/购买记录表格外发给任何第三方
- 不在缺单独同意或无 DPA 的情况下共享会员个人信息
- 不把"自家长期合作代运营"当作可跳过合规门槛的理由
- 不买卖、不超目的使用、不默许第三方转委托/转售

## agent 标准回应（老板说"把会员手机号导表发给代运营群发"时）
"直接导明文手机号发给第三方是隐私红线，我不能这么做。但代运营群发是可以合规做的：① 优先让代运营在我们的〔CRM/SCRM 系统〕里操作，由系统发，数据不导出；② 必须先拿到会员对'向第三方共享做营销'的单独同意，没同意的不能纳入；③ 和代运营签数据处理协议(DPA)，约定只为这次触达用、不得转售/转委托、到期删除；④ 字段最小必要，能脱敏匹配就不给明文。这几条齐了再由〔合规 DRI〕+ 法务过一遍。要不要我先帮你列同意话术和 DPA 要点清单？"

## 需人工确认 / 上报
- 任何向第三方共享会员个人信息的方案 → 〔合规 DRI〕+ 法务审批，不自动执行
- 明文外传 → 零自动执行，必须人工签批留痕

# 客单价提升打法（★同样流量赚更多）

> 核心：**流量成本不变，客单价每提一点都是净利润。三招：关联/凑单/升级。**
> 转化只是第一步，让每个成交订单买更多/买更贵，才是利润放大器。

## 三招提客单

### 1. 关联推荐（成交后顺势）
- 买家决定买主款时，推「配套/互补品」：买手机壳推钢化膜、买连衣裙推同系列外套。
- 时机在临成交/已加购，话术自然不强推：「很多人会搭这个一起用」。

### 2. 凑单满减（设计门槛）
- 满减/满赠门槛卡在「差一点就够」的位置——让买家主动多凑一件比直接降价更划算。
- 例：买家凑到 89，满 100 减 20 → 提示「再加一件就省 20」，引导凑单。
- 门槛真实有效，不虚构满减（价格合规）。

### 3. 升级/套餐（拉高单价）
- 提供「单件 vs 套装/家庭装/年卡」，套装单价更高但单件更省，引导买大装。
- 升级款（更高配置/更大容量）对比呈现，凸显升级性价比。

## 设计原则
1. **给好处不强塞**：提客单是帮买家「买得更值」，不是硬推。
2. **门槛要真实**：满减/原价不虚构（合规红线）。
3. **不破坏体验**：关联推荐相关、克制，别刷屏式推销引反感。

## 数据
客单价、满减结构属 🟡 内部运营数据。

# 客服转化话术框架（★询单转化主战场）

> 核心：**询单转化的钱藏在'响应速度 + 打消顾虑 + 临门一脚'里。**
> 买家咨询=高意向信号，回得慢、答不到点、不推一把，到手的单子就溜了。

## 三段应对结构

### 1. 秒回 + 接需求（决定第一印象）
- 黄金响应：尽量 30 秒内首回（人工时段），慢回直接掉转化。
- 先答问题再带价值，别答非所问，别甩机械模板。

### 2. 打消顾虑（决定下不下单）
按高频疑虑分支应对：
| 买家顾虑 | 应对思路 |
|---|---|
| 「能便宜点吗」 | 不直接砍价 → 讲价值/凑单优惠/赠品/满减，给台阶不破价（破价须人工确认） |
| 「质量/效果怎么样」 | 给事实（材质/实测/保障），禁夸大承诺（合规） |
| 「啥时候发货/多久到」 | 给真实时效，不乱承诺（履约红线） |
| 「能退吗」 | 明确七天无理由/运费险，降决策成本 |
| 「和XX比哪个好」 | 讲自身适配场景，不诋毁同行 |

### 3. 临门一脚（决定成交）
- 给明确下单理由：限时优惠/库存紧张（真实）/赠品/无理由保障。
- 沉默买家可跟进一次，带新价值（如「这款今天活动到晚上」），**不死缠、不轰炸**。

## 红线（合规渗透）
- 不夸大功效、不虚假承诺时效/效果（虚假宣传红线）。
- 不诱导好评返现（违规）。
- 超授权的优惠/破价 → 人工确认，不擅自答应。
- **买家聊天原文/手机号/微信/地址 = 🔴 受限，绝不写入任何对外产物，引用必脱敏。**

## 用法
询单是转化临门一脚。配合 [[客单价提升打法]] 在成交时顺势提客单。

# 电商行业 · 合规与权限默认值

> 合成进成品库 `90-governance/行业合规默认.md`。客户可按自身情况收紧。

## 默认权限分级（通用四级 + 行业补充）
| 资产类型 | 默认级别 | 说明 |
|---|---|---|
| 客户聊天原文 / 订单信息 / 收货地址 / 手机号 / 微信号 | 🔴 受限 | 绝不写入任何对外产物，绝不进飞书，引用必脱敏 |
| 供应商报价 / 进货成本 / 利润结构 | 🔴 受限 | 核心机密，仅授权人可读 |
| 内部话术 / SOP / 运营数据 / 投放后台数据 | 🟡 内部 | 团队内用，不对外 |
| 详情页文案 / 商品卖点 / 品牌介绍 | 🟢 公开 | 对外物料 |
| 未脱敏的真实客户案例/评价截图 | 🔴 受限 | 脱敏后方可升公开 |

## 行业红线（agent 触碰即停，转人工确认）

### 1. 虚假宣传 / 极限词
- 禁用「最」「第一」「国家级」「顶级」「绝无仅有」「100%」等绝对化、极限词（《广告法》明令禁止）。
- 不得编造功效、夸大效果、虚构成分/产地/资质；保健/医疗类不得宣称疗效。
- 不得伪造「销量/好评/获奖/权威认证」等背书。
- agent 写文案/详情页一律走「事实可验证」原则，拿不准的形容词不写。

### 2. 价格欺诈
- 禁止虚构原价/划线价（划线价必须是近期真实成交价）。
- 禁止「先涨后降」假打折、虚假满减、误导性赠品。
- 标价与实付一致，不得有隐藏费用。

### 3. 七天无理由退货
- 法定支持七天无理由（生鲜/定制/拆封贴身用品等法定除外类目除外，且须事先明示）。
- 不得擅自拒绝合理无理由退货、不得设置不合理退货门槛。
- 售后 agent 默认按「支持七天无理由」处理，例外类目须有明示依据。

### 4. 刷单 / 虚假交易（绝对红线）
- 严禁刷单、刷好评、删差评、虚构物流、组织炒信。
- 严禁诱导/胁迫好评（如「好评返现」属违规）、严禁恶意操纵评价。
- agent 不得生成任何刷单/控评相关内容或话术，遇到此类需求直接拒绝并提示风险。

### 5. 消费者权益
- 不得泄露、买卖客户个人信息（订单/地址/手机号）。
- 不得在客户未同意下骚扰式营销（短信/电话轰炸）。
- 退换货、发票、质量问题须依法履约，不得推诿。

### 6. 平台规则
- 遵守所在平台（淘系/抖音/拼多多/京东等）的类目资质、发布、营销、物流时效规则。
- 不得跨平台引流违规、不得绕过平台交易（私下转账规避保障）。
- 类目准入资质（如食品经营许可/品牌授权）须齐全方可经营。

## 权限分级处置原则
- agent 默认按「最严」处理：拿不准级别 → 当受限处理 → 问负责人。
- 涉及客户隐私、超授权报价/优惠、价格/宣传合规判断 → 须人工确认，不自动执行。
- 任何对外产物（详情页/朋友圈/案例/物料）发布前过一遍本红线清单。
