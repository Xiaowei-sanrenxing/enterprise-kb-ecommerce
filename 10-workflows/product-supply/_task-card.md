# 任务卡 · 选品与供应链（product-supply）

> Agent 操作总台。接到"评估新品/定货源/算备货/巡库存"类任务，从这里开始。

## 🎯 目标(Goal)
把一个候选品，从评估推进到「上/不上」的决策，定好货源与首批备货量，并维持库存健康——全程不踩三大坑（拍脑袋选品/负毛利硬上/断货掉权重）。

## 📥 上下文(Context) —— 动手前必读全
1. [`context/选品三维评估模型.md`](context/选品三维评估模型.md) — 需求×竞争×利润打分定生死
2. [`context/备货与库存健康线.md`](context/备货与库存健康线.md) — 备多少、什么时候补、怎么清滞销
3. 〔货源/供应商库〕 — 报价、起订量、工期（真实库含，🔴受限）
4. 〔类目数据工具〕 — 搜索量/竞争/趋势（真实库接生意参谋等）

## 🪜 指令(Instructions)
1. 拉候选品的需求/竞争/利润数据 → 过三维模型打分
2. 任一轴 <3 或毛利 <20% → 毙掉或提改造方案（换货源/找差异化）
3. 过关品 → 比价定货源（核对资质/起订量/工期）
4. 按备货公式算首批量（新品小批试，别一把梭）
5. 上架后每日巡库存健康线 → 预警即补、断货风险降速、滞销果断清
6. 每个决策记录依据与结果 → 回流复盘

## 🛠 工具(Tools)
〔类目数据工具〕 · 〔货源/供应商库（受限）〕 · 〔库存/ERP 读写〕 · 三维打分模板

## 🔒 权限(Permission)
- 可读：选品模型、备货规则、公开类目数据
- 🔴 受限：供应商报价/进货成本/利润结构 → 仅授权人，绝不外发
- 需人工确认：大额首单备货、新供应商合作、清仓甩货定价

## ✅ 验收(Acceptance)
- 每个上架决策有三维打分依据 ｜ 毛利<20% 未硬上 ｜ 库存无意外断货/超期滞销 ｜ 成本数据未外泄
- 详见 evals.md

## 🔄 自我进化接口
每个品记录"打分 → 实际表现（爆/平/滞销/退货率）" → 回流校准三维模型阈值 → 下次选品更准。
