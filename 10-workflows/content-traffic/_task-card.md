# 任务卡 · 内容与流量（content-traffic）

> Agent 操作总台。接到"写/优化详情页、做直播脚本、做投放计划"类任务，从这里开始。

## 🎯 目标(Goal)
为一个商品搭好转化承接（详情页/直播）并把对的流量引进来——全程不踩三大坑（详情页结构乱/承接力差就硬投流/宣传踩合规红线）。

## 📥 上下文(Context) —— 动手前必读全
1. [`context/详情页黄金三屏.md`](context/详情页黄金三屏.md) — 首屏抓人/中屏说服/尾屏逼单
2. [`context/流量结构与投放红线.md`](context/流量结构与投放红线.md) — 先承接后放量、测款节奏、ROI 守线
3. `../../90-governance/行业合规默认.md` — 极限词/虚假宣传/价格红线（写文案必读）
4. 〔投放后台/数据工具〕 — 点击率/转化率/ROI（真实库接）

## 🪜 指令(Instructions)
1. 接任务先判属哪类：详情页 / 直播脚本 / 投放计划
2. 详情页 → 按黄金三屏搭结构，卖点翻成"好处"，过合规自检
3. 直播脚本 → 同样三段逻辑（抓人/说服/逼单），过合规
4. 投放 → 先查承接力（转化率达健康线没），不达标先优化别加投
5. 新品投放 → 小预算测点击/转化/ROI，三关过再放量
6. 监控 ROI/库存：<盈亏平衡止损、断货降速
7. 记录素材/页面表现 → 回流沉淀爆款结构

## 🛠 工具(Tools)
〔详情页/页面编辑〕 · 〔投放后台读写〕 · 〔数据工具〕 · 合规自检清单 · 三屏/三段模板

## 🔒 权限(Permission)
- 可读：内容模板、投放规则、合规默认
- 🟡 内部：投放后台数据/ROI/成本 → 不对外
- 🔴 受限：客户评价原文/截图 → 引用必脱敏，禁刷单控评
- 需人工确认：大额投放预算、超授权优惠、拿不准的合规表述

## ✅ 验收(Acceptance)
- 详情页过三屏结构 + 合规自检 ｜ 投放前查过承接力 ｜ 新品走测款再放量 ｜ 无极限词/虚假/虚构价 ｜ 评价脱敏无刷单
- 详见 evals.md

## 🔄 自我进化接口
每个页面/素材记录"结构 → 转化率/ROI 表现" → 回流沉淀高转化结构模板 → 下次直接复用爆款骨架。
