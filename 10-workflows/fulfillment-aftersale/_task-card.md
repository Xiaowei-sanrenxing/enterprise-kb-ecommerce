# 任务卡 · 履约与售后（fulfillment-aftersale）

> Agent 操作总台。接到"处理发货/退换/物流异常/评价管理"类任务，从这里开始。

## 🎯 目标(Goal)
把一单从发货到售后处理完，依法快处退换、把退货数据变成改进燃料、把差评变挽回机会——全程不踩三大坑（推诿违规拒退/刷单删评掩盖问题/退货不归因白退）。

## 📥 上下文(Context) —— 动手前必读全
1. [`context/退换货处理标准流程.md`](context/退换货处理标准流程.md) — 接诉/判类型/给方案/归因四步
2. [`context/退货归因与评价管理.md`](context/退货归因与评价管理.md) — 退货回流闭环 + 评价合规处理
3. [`context/恶意差评合规申诉举报路径.md`](context/恶意差评合规申诉举报路径.md) — 守住不删评红线后，恶意/同行/敲诈差评的正向申诉举报路径
4. `../../90-governance/行业合规默认.md` — 七天无理由/刷单/删评红线
5. 〔订单/物流系统〕 — 发货时效/物流状态（真实库接，🔴受限含订单信息）

## 🪜 指令(Instructions)
1. 发货：按时效发货，异常（缺货/物流慢）主动预警买家
2. 售后接诉 → 先共情认问题 → 判类型（无理由/质量/物流/错漏发/异常）
3. 依法给方案：七天无理由默认通过、质量问题优先换退担运费
4. 超授权补偿/特殊处理 → 人工确认
5. 每笔退货记真实原因 → 回流对应工作流（质量→选品、不符→详情页…）
6. 中差评第一时间真诚联系解决，自然争取修改（禁胁迫/利诱/删评/刷单）
7. 退货率/差评异常 → 拉归因分布定位根因

## 🛠 工具(Tools)
〔订单/物流系统读写（受限）〕 · 〔售后工单系统〕 · 退货归因表 · 评价管理后台

## 🔒 权限(Permission)
- 可读：售后流程、归因规则、合规默认
- 🔴 受限：买家订单信息/收货地址/手机号/聊天 → 处理中不外泄，归因记录脱敏
- 需人工确认：超标准补偿、恶意退货定性、争议升级

## ✅ 验收(Acceptance)
- 七天无理由依法处理不违规拒退 ｜ 退货全部归因回流 ｜ 中差评真诚处理无刷单删评 ｜ 无好评返现 ｜ 订单隐私零外泄
- 详见 evals.md

## 🔄 自我进化接口
退货原因/差评内容按周汇总 → 找占比最高根因 → 回流对应工作流修 → 下次同类问题不再发生。
