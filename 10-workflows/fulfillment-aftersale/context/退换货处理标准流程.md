# 退换货处理标准流程（★售后核心 + 合规高压区）

> 核心：**依法快处、息事宁人、把退货变复购机会——不推诿、不违规拒退。**
> 售后处理得好，差评变好评、退货客变回头客；处理砸了，差评+投诉+平台扣分三连。

## 退换货处理四步

### 1. 接诉 + 判类型
| 类型 | 处理原则 |
|---|---|
| 七天无理由 | 法定支持，默认通过（法定除外类目须事先明示才可拒，见合规） |
| 质量问题 | 优先换/退 + 必要补偿，承担来回运费 |
| 物流问题（破损/丢件）| 查物流凭证，补发/退款，向承运方追责 |
| 错发/漏发 | 立即补发/退差，态度先认错 |
| 恶意/异常退货 | 留证据，按平台规则处理，不擅自定性 |

### 2. 快速响应 + 安抚
- 先共情认问题，再给方案，别先甩责任。
- 不推诿、不设不合理门槛（合规红线）。

### 3. 给方案 + 闭环
- 能退/换/补就利落给，必要时小额补偿挽留。
- 超授权补偿/特殊处理 → 人工确认。

### 4. 归因记录（接售后归因闭环）
- 每笔退货记真实原因 → 回流（见 [[退货归因与评价管理]]）。

## 合规红线（绝对不可碰）
- **不得违规拒绝七天无理由**（法定除外类目须事先明示）。
- **不得拿处理售后胁迫/交换删差评**（如「退款可以，但你得删差评」属违规）。
- **不得诱导好评返现**。
- **不得拖延/推诿法定售后义务。**

## 隐私红线
- 买家订单信息/收货地址/手机号 = 🔴 受限，处理过程不外泄，记录归因时脱敏。

## 用法
售后是履约的兜底。处理完务必把退货原因喂给 [[退货归因与评价管理]] 形成改进闭环。
